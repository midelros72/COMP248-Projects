Project Overview

Project Type: Medical Information Retrieval System

Core Technology: RAG (Retrieval-Augmented Generation)

Summary: An intelligent multi-agent system that processes health-related queries using a combination of specialized AI agents. The system retrieves relevant medical information from a curated knowledge base and provides accurate, source-backed responses through a clean Streamlit interface.


Key Features:

  • ü§ñ Multi-Agent Architecture: Planner, Search, Summarize, and Reflective agents

Data Processing Pipeline

  1. Text Extraction: Medical documents processed and chunked
  2. Vectorization: Sentence transformers create embeddings
  3. Storage: FAISS index for fast similarity search
  4. Retrieval: Top-k relevant chunks for each query

Workflow Phases

1. üéØ Planning Phase

Planner Agent analyzes the user query and creates a structured research plan

2. üîç Search Phase

Search Agent queries the vector database and retrieves relevant medical information

3. üìù Synthesis Phase

Summarize Agent processes retrieved chunks into a coherent, medically accurate response

4. ü§î Reflection Phase

Reflective Agent validates the response quality and suggests improvements

Sample Query Flow


# User Query: "What are the symptoms of Type 2 diabetes?"

# 1. Planner Agent Output:

Research Plan: Type 2 Diabetes Symptoms

2. Search for early warning signs  
3. Extract diagnostic criteria
4. Summarize in accessible language

# 2. Search Agent Retrieval:
relevant_chunks = vector_search("type 2 diabetes symptoms")

# 3. Summarize Agent Response:
response = llm_generate(plan + relevant_chunks)
# Produces: Comprehensive symptom overview with sources

# 4. Reflective Agent Validation:
validation = assess_medical_accuracy(response)
# Ensures: Medical disclaimers and accuracy

LLM Integration Strategy

Local-First Approach

  • Privacy: All processing happens locally via Ollama
  • Cost-Effective: No API fees for medical queries
  • Reliability: Offline capability for critical health information

Model Configuration


# Environment-based LLM selection
USE_OLLAMA = True
OLLAMA_MODEL = "mistral:latest"

def get_llm_config():
    if USE_OLLAMA:
        return Ollama(model=OLLAMA_MODEL)

        return OpenAI(api_key=OPENAI_API_KEY)
